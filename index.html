Logic Explanation

The app uses one array called lootItems to store all the loot. This array is created at the top of the file, so it stays available the whole time the page is open. Every function can read and update the same array. Each loot item is a plain object with three parts: a name (text), a value (number), and a quantity (number). Quantity was added so one item can represent multiple copies without needing a separate entry for each one.

When the user clicks "Add Loot," the addLoot() function checks a few things: the name can't be empty, the value has to be a real number, the value can't be negative, and the quantity has to be 1 or greater. If everything is okay, the item is added to the array and updateUI() runs immediately.

updateUI() is the only place where totals are calculated and the screen is updated. It loops through the array, multiplies each item's value by its quantity to get the total, rebuilds the loot list on screen, and then checks if the state is valid. If the array has items and the party size is a valid number, the Split button becomes enabled and the results are shown. If either of those is missing, the button stays disabled and the results stay hidden. Removing an item calls removeLoot() which uses splice() to delete it from the array, then calls updateUI() again. The array is always the source of truth.
