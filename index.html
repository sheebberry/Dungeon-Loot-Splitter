Logic Explanation

The app uses one list called lootItems to store all the loot. This list is created at the top of the file, so it stays available the whole time the page is open. Every button can use and update the same list. Each loot item has two parts: a name (text) and a value (number).

When the user clicks "Add Loot," the addLoot() function checks a few things: the name can’t be empty, the value has to be a real number, and it can’t be negative. If everything is okay, the item is added to the list. Then renderLoot() runs. It uses a loop to go through all the items, show them on the screen, and calculate the total value.

When the user clicks "Split Loot," the splitLoot() function adds up all the loot again. It checks that the party size is valid and that there is loot to split. If everything is valid, it divides the total amount evenly.
