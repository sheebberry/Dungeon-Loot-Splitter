Logic Explanation

The app stores all loot in one array called lootItems Each item is an object with a name, value, and quantity. This array is created at the top of the file so all functions can access it, making it the main source of truth for the app.

When the Add Loot button is clicked, addLoot() validates the inputs to make sure the name is not empty, the value is a valid non-negative number, and the quantity is at least 1. If the inputs are valid, the new loot object is added to the array and updateUI() runs.

updateUI is responsible for recalculating totals and updating the screen. It loops through the array to calculate total loot (value × quantity), rebuilds the loot list, and checks if the app state is valid. If loot exists and the party size is valid, the Split button is enabled and results are shown.

Each loot item has a Remove button that calls removeLoot(). This function removes the item from the array using splice() and then calls updateUI() so the interface updates to match the current data.
